# Тренажёр по обществознанию

Сайт с тестами по программе вступительного испытания по обществознанию (МГЮА): 7 разделов, 245 вопросов, вопросы с 1, 2 или 3 правильными вариантами ответа, частичный зачёт за неполный ответ (как на реальном экзамене), сохранение прогресса, разбор ошибок.

Это полностью статический сайт — просто HTML, CSS и JavaScript. Никакого сервера, базы данных или установки зависимостей не требуется. Весь прогресс ученика хранится в браузере (localStorage), то есть данные привязаны к конкретному браузеру на конкретном устройстве.

## Структура проекта

```
quiz-site/
├── index.html          — главная страница (разделы + общий прогресс)
├── quiz.html            — страница прохождения теста
├── review.html          — страница "Мои ошибки"
├── css/style.css         — стили
├── js/
│   ├── storage.js        — сохранение прогресса в localStorage
│   ├── scoring.js         — перемешивание вопросов и подсчёт баллов
│   ├── app.js              — логика главной страницы
│   ├── quiz.js              — логика прохождения теста
│   └── review.js             — логика страницы ошибок
└── data/
    ├── meta.js               — список разделов (названия, описания)
    └── section1.js…section7.js — банки вопросов по разделам
```

## Как запустить локально

Никакой установки не требуется — это обычные файлы.

**Вариант 1 (самый простой).** Просто откройте `index.html` двойным щелчком — сайт откроется в браузере и будет работать. (Специально сделано так, что вопросы подключены как обычные `<script>`-файлы, а не через `fetch`, — поэтому не возникает ошибок браузера при открытии файлов напрямую с диска.)

**Вариант 2 (если хотите открыть через локальный сервер).** Из папки `quiz-site` выполните одну из команд и откройте `http://localhost:8000`:

```bash
# если установлен Python 3
python3 -m http.server 8000

# либо через Node.js (npx)
npx serve .
```

## Как загрузить на GitHub и открыть сайт по ссылке (GitHub Pages)

1. Создайте новый репозиторий на GitHub (например, `oe-quiz`), можно пустой, без README.
2. На своём компьютере, в терминале, перейдите в папку `quiz-site` и выполните:

   ```bash
   git init
   git add .
   git commit -m "Тренажёр по обществознанию"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГИН/oe-quiz.git
   git push -u origin main
   ```

3. Зайдите в репозиторий на GitHub → **Settings** → **Pages** (в левом меню).
4. В разделе **Build and deployment** → **Source** выберите **Deploy from a branch**.
5. В **Branch** выберите `main` и папку `/ (root)`, нажмите **Save**.
6. Через 1–2 минуты сайт будет доступен по адресу:

   ```
   https://ВАШ_ЛОГИН.github.io/oe-quiz/
   ```

Готово — ссылку можно давать ученикам. Дальнейшие изменения (например, добавление новых вопросов) публикуются автоматически при каждом `git push`.

## Как добавить или изменить вопросы

Откройте нужный файл `data/sectionN.js` — это обычный JS-массив объектов вида:

```js
{
  id: "s1-01",
  topic: "Общество и природа",
  question: "Текст вопроса...",
  options: ["вариант а", "вариант б", "вариант в", "вариант г"],
  correct: [0],       // индексы правильных вариантов (0 = а, 1 = б, 2 = в, 3 = г)
  explanation: "Пояснение, показывается после ответа."
}
```

Правильных вариантов может быть от 1 до 3 (укажите нужные индексы в `correct`). ID вопросов должны быть уникальны в пределах всего сайта.

## Как устроен подсчёт баллов

За вопрос с несколькими правильными ответами применяется частичный зачёт (как на реальном экзамене):

```
балл_за_вопрос = (число верно выбранных − число неверно выбранных) / (сколько ответов верно всего)
```

Результат не может быть меньше 0. Например, если правильных ответов два, а ученик выбрал только один (без ошибок) — он получает 50% балла за этот вопрос. Если выбрал один верный и один неверный — балл обнуляется.

## Прогресс и сброс

- Прогресс по каждому вопросу — это лучший результат из всех попыток (не портится, если позже ответить хуже).
- Раздел «Мои ошибки» показывает вопросы, где *последний* ответ был не полностью верным — стоит порешать их ещё раз.
- Кнопка «Сбросить весь прогресс» на главной странице полностью очищает сохранённые данные в этом браузере.
- Если открыть сайт в другом браузере или на другом устройстве — прогресс не переносится (данные локальные). При необходимости синхронизации между устройствами понадобится добавить сервер/аккаунты — сейчас сайт спроектирован как максимально простой и не требующий бэкенда.
